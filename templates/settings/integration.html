Integration & Automation

What does this feature do?

When enabled, a Resolution dropdown is injected into every action modal (Ignore, Bulk Ignore, Bulk Suppress, Create Exception, Create Suppression). The user must select a resolution category before the action can be submitted.

After the action succeeds, a webhook POST is fired to the configured URL (e.g. a Logic App endpoint) so your external ticketing system can automatically close or update the corresponding ticket. If no webhook URL is configured, the resolution dropdown still appears and the value is validated — the webhook simply does not fire.

Webhook payload (JSON)

Every webhook delivery sends a JSON body with these fields:

action Event type: case_ignored, exception_created, suppression_created, bulk_ignore, or bulk_suppress
rule_id The Wazuh rule ID involved in the action
rule_description Human-readable description of the rule
notes Free-text notes entered by the analyst
resolution Selected resolution category (e.g. "False Positive")
username The platform user who performed the action
timestamp UTC timestamp in ISO 8601 format
case_id Alert case ID (single actions only)
case_ids Array of case IDs (bulk actions only)
rule_ids Array of rule IDs (bulk actions only)
field_name Exception field path (exceptions only)
field_value Exception field value/pattern (exceptions only)
customer Customer identity value from the alert case (requires Customer Field configured)
user_email Email address of the user who performed the action

Tip: To include customer identity in webhook payloads, configure the Customer Field in Settings > General.

Example payload 
{
  "action": "case_ignored",
  "rule_id": "60204",
  "rule_description": "Windows Logon Success",
  "notes": "Known admin activity on jump server",
  "resolution": "False Positive",
  "username": "jsmith",
  "timestamp": "2026-04-30T14:23:00",
  "case_id": 42
}
Configuration summary
  • Webhook URL — The HTTP(S) endpoint that receives the POST (e.g. Azure Logic App trigger URL).
  • Auth Type — Bearer Token sends Authorization: Bearer <token>. Custom Header lets you specify a custom header name (e.g. X-Api-Key).
  • Timeout / Retry — Each delivery attempt times out after N seconds. Failed attempts are retried up to the configured count.
  • Resolution Categories — The dropdown options shown to analysts. Customize them to match your ticketing workflow.
  • Webhook Events — Toggle which action types fire a webhook. Disabled events still require a resolution selection but skip the HTTP call.
  • Recent Deliveries — A log of all webhook attempts with HTTP status, success/failure, and any error messages for debugging.
